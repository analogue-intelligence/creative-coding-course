Exercises week 1

3.1 Variables

3.1.1 Shapes and Variables

Draw a circle such that the x-position, y-position and the radius are all defined using variables

3.1.2 Constants

Draw a circle just as we did in the 3.1.1, however now set the radius as a constant. Try changing the value of radius after definition and see what happens.

A constant is ... well constant, it cannot be redefined after initialisation.

3.1.3 Global/Local variables

Create a circle again using variables, however this time define the radius variable inside setup() - in another function.

What happens? the radius variable is defined within another function and is said to be out of scope so it cannot be used. Variables defined outside of any function are said to be globally defined, while variables defined within a function or {} braces (a block) are said to be defined locally.

3.1.4 Draw Loop

Draw a circle using variables just as we did in the first exercise but with radius initialised as 0, however in this case increment the radius by 0.1 in the draw loop.

What is happening? Since draw() is an infinite loop, the radius is being incremented at every new frame which makes the circle seem to constantly enlarge.

3.1.5 System Variable

A system variable is a variable name which is predefined in the system, draw a shape using the system variables 'height', 'width' and 'frameCount'

3.1.6 Random

Draw a shape, with random size, position and colour. Make sure that the shape is drawn whenever the mouse is pressed. Make sure to create variables for each random element of the shape.

3.2 Conditionals

3.2.1 If

Draw the left side of the screen red whenever the mouse is on the left side of the screen

3.2.2 If-Else

Do the same as above, however now when the mouse is on the top side of the screen draw the top side blue.

What happens in this case? Does the program draw the left side or the top when on the top-right section of the screen. To help visualise what is happening, try drawing two lines, one which goes through the horizontal center, while another which goes through the vertical center. The priority of which conditional statement is dependent on the order of the conditionals.

3.2.3 Else

Do the same as above again, and now if the mouse is not on the left, and not at the top of the screen, then draw the bottom-right side of the screen green.

3.2.4 Nested If

Using a nested if-statement, first check if the mouse is on the left side of the screen, and then check if the mouse is on the bottom side of the screen. If both conditions pass then colour the bottom-left part of the screen any colour.

What do you think is the difference when neither if-statements is nested? In this case both conditionals will execute independant of each other.

3.2.5 Boolean Operations

Try to combine the two if-statements above using boolean operations (and, or, not). Also try to create more conditional statements with the boolean operations.

3.2.6 While loop

Using a while loop, draw lines across the screen. As a hint, define the spacing between the lines using a variable and also the condition in the while loop should include the total width of the screen so as to know when to stop iterating. Continuously increment a variable with the spacing and draw a line up until the width of the screen has been reached

Create the same program as the one in 3.2.6 using a for loop

3.2.8 Grid

Create a grid using a nested for loop, to create the grid have the first for loop focus on the rows and the second for loop focus on the columns (or vice-versa) and then create a rectangle at each row-column index
